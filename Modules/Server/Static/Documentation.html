About the Frizz Project
                    The frizz project was designed as a competitor to APACKETS. A better, cheaper offline 
                    and more accurate and reliable NFAT than apackets. This project 
                    includes support for multiple authentication types and protocols. 
                    The frizz project is built to ensure that most red teamers can parse millions of packets 
                    in seconds and have them displayed on an amazing web interface. 
                    With the power of Go, Crystal and C++ the server is guarenteed to provide you with an amazing,
                    modern and sleak interface design.

                    Frizz comes with many tools which allow you to view and manipulate your way around packet data. 
                    You may be asking can I not do this with wireshark? The point is 
                    not about what you can or can not use it for but how it gets it done. 
                    Using modern day algorithms and custom implimentations they are designed to parse as much possible 
                    information it can to ensure your experience.

Frizz Utilities -> PCAP masher
                    Frizz has many utilities the main ones which include packet mashing which is a 
                    utility in the Packet Masher tab 

                    this tool allows you to take a giant list of paths to pcap
                    files and mash every packet into one giant output file which allows you to parse it 

                    all at once. This helps with rendering it all at once instead of rendering it one by one
Frizz Utilities -> PCAP Line finder
                    in tab Packet Extractor is a utility that is really in here for 
                    fun but it allows you to take a given pcap file and search for a line inside of the file then output it to a 

                    output pcap file. The reason this was in here is to help during security research in 
                    filtering and exposing a vulnerability in a packet capturing 

                    application. This one is quite simple
Frizz Utilities -> PCAP uploader 
                    in tab Parse New is a tool that simply allows you to upload a new pcap file to the server 

                    without having to leave and restart the server.
Tab | Application Information
                    the tab Application Information is a data table 
                    and box of all of the information about the frizz project such as app version,
                    app name, app binary, files etc. This information is only here for developers 
                    and users that may be running professional licenses.
Tab | Server Information
                    the tab Server Information is a data table 
                    and data row of everything about the current server which includes the binary,
                    version, title, All the server URL's, All server Ports, All server Hosts, all server 
                    import list for CSS, supported browsers 

                    this information will help give you a better UX/UI experience.

Tab | JSONDB
                    the tab JSON Server File Is a bunch of 
                    code / syntax boxes which include json data. The json data you see 
                    is the data that frizz finds within your packet capture. This is more or 
                    less a debugging things for development and to where the dev(s) can view

Tab | Application Information
                    the tab Info This Server Needs Alot of people 
                    will get mad at this but welp you downloaded the tool so your welcome. This is a tab 
                    of all of your hardware information such as 

                    * CPU information -> Model, Model name, index number, cache, cache size, speed, type, 
                                        Micronode, Step, Physical ID, Family number, Vendor ID
                    
                    * OS / Kernel information -> OS Family, OS Host, OS Version, OS name, OS Type, Kernel type
                                        Kernel version, Kernel family, Kernel architecture
                    
                    The reason this information is collected is because in later versions of frizz this information 
                    will be used to leverage the amount of threads that are started or ran on your CPU and the amount of 
                    server connections are being made at once. This allows frizz to work with your hardware to make frizz's 
                    server as fast and performant as possible while providing an amazing UI/UX design.
